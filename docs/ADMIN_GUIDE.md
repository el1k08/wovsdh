# Admin Panel Guide — Руководство для сотрудников

Полное руководство по использованию админ-панели для управления слотами записей.

## 1. Доступ к админ-панели

### 1.1 URL админ-панели

```
https://your-domain.com/admin
```

Для локальной разработки:

```
http://localhost:3000/admin
```

### 1.2 Авторизация

1. Открыть URL админ-панели
2. Увидите форму ввода пароля
3. Ввести значение переменной `ADMIN_SECRET_KEY` (выдаёт администратор)
4. Нажать **Вход**

**Примечание:** пароль хранится только в сессии браузера, при перезагрузке нужно будет ввести заново.

## 2. Интерфейс админ-панели

### 2.1 Выбор студии и фильтры

В верхней части страницы:

- **Студия** — dropdown с выбором:
  - Rishon LeZion
  - Ashdod
- **От** — дата начала периода (формат: YYYY-MM-DD)
- **До** — дата конца периода (формат: YYYY-MM-DD)
- **Загрузить** — кнопка для фильтрации слотов

**Пример:**
```
Студия: Rishon LeZion
От: 2025-06-01
До: 2025-06-30
→ Загрузить
```

Будут показаны все слоты в студии Rishon LeZion за июнь 2025.

### 2.2 Таблица слотов

Результаты отображаются в таблице с колонками:

| Колонка | Описание | Цвет |
|---------|---------|------|
| **Дата** | Дата слота (формат: DD.MM.YYYY) | — |
| **Время** | Время начала и окончания (HH:mm - HH:mm) | — |
| **Студия** | Название студии | — |
| **Статус** | Статус слота (см. ниже) | Цветной |
| **Клиент** | Имя и фамилия клиента (если занят) | — |
| **Действия** | Кнопки для управления | — |

### 2.3 Статусы слотов

Слоты имеют три возможных статуса:

| Статус | Цвет | Описание | Действие |
|--------|------|---------|---------|
| **Available** | Зелёный | Свободный слот, может быть забронирован | Можно удалить или заблокировать |
| **Booked** | Красный | Занят клиентом, ожидает подтверждения | Показывает имя клиента, можно удалить |
| **Blocked** | Серый | Заблокирован (обеденный перерыв, техническое обслуживание) | Можно разблокировать (удалить) |

## 3. Создание слотов

### 3.1 Форма создания

В верхней части страницы находится форма **Создание слотов**:

**Поля:**

- **Студия** — выбрать студию (Rishon LeZion или Ashdod)
- **Дата** — дата для слотов (формат: YYYY-MM-DD)
- **Время начала** — начало рабочего дня (формат: HH:mm, например 09:00)
- **Время окончания** — конец рабочего дня (формат: 17:30)
- **Длительность записи, мин.** — продолжительность одной процедуры (например, 60 мин для маникюра)

**Пример:**

```
Студия: Rishon LeZion
Дата: 2025-06-01
Время начала: 09:00
Время окончания: 17:30
Длительность записи: 60 мин
```

Система создаст слоты:

```
09:00 - 10:00
10:00 - 11:00
11:00 - 12:00
12:00 - 13:00
(перерыв на обед предполагается отдельно)
13:00 - 14:00
...
16:30 - 17:30
```

### 3.2 Нажать "Создать слоты"

После заполнения формы нажать кнопку **Создать слоты**.

**Результат:**

- Система создаст все слоты на указанную дату
- Появится сообщение: `Создано 8 слотов, пропущено 0`
- Таблица обновится и покажет новые зелёные слоты

### 3.3 Обработка конфликтов

Если на дату уже существуют слоты:

- Система пропустит уже существующие
- Создаст только новые (непересекающиеся)
- Сообщение покажет: `Создано X слотов, пропущено Y`

**Пример:**
```
Создано 6 слотов, пропущено 2
```

Означает, что 6 новых слотов создано, 2 уже существовали.

## 4. Управление слотами

### 4.1 Удаление свободного слота

1. Найти зелёный слот в таблице (статус: Available)
2. Нажать кнопку **Удалить** в колонке Действия

**Подтверждение:** система попросит подтвердить удаление

**После удаления:** слот удаляется из БД, статус сообщит: `Слот удалён успешно`

### 4.2 Удаление занятого слота

1. Найти красный слот (статус: Booked)
2. Нажать **Удалить**

**Важно:** при удалении занятого слота:

- Запись клиента отменяется (статус → CANCELLED)
- Клиент получит email об отмене
- Событие в Google Calendar будет удалено

**Используйте этот функционал только в крайних случаях** (например, закрытие студии на день).

### 4.3 Удаление заблокированного слота (разблокировка)

1. Найти серый слот (статус: Blocked)
2. Нажать **Удалить**

**После удаления:** слот удаляется, это работает как разблокировка.

### 4.4 Создание блокированных слотов

Чтобы создать обеденный перерыв или закрыть студию:

1. Создать слоты с **Длительностью записи** совпадающей с длительностью перерыва
2. Вручную обновить статус в Supabase:

```sql
UPDATE slots 
SET status = 'blocked' 
WHERE studio_id = 'rishon' 
  AND start_at >= '2025-06-01 12:00:00' 
  AND start_at < '2025-06-01 13:00:00';
```

Или через админ-панель (в будущем версии).

## 5. Telegram-интеграция

### 5.1 Как работает?

Когда клиент делает запись через сайт:

1. Создаётся запись в БД со статусом **PENDING**
2. Все авторизованные сотрудники получают сообщение в Telegram боте
3. Сообщение содержит кнопку **"Подтвердить"**

### 5.2 Структура сообщения в Telegram

```
📅 Новая запись!

Клиент: Анна Иванова
Телефон: +972501234567
Email: anna@example.com
Студия: WOVSDH Nails Rishon
Дата и время: 1 июня 2025, 08:00 - 09:00

[Подтвердить] [Отменить]
```

### 5.3 Подтверждение записи

Нажать кнопку **Подтвердить** в Telegram:

1. Запись обновляется → статус CONFIRMED
2. Событие создаётся в Google Calendar
3. Клиент получает email с подтверждением и .ics файлом
4. Кнопка в Telegram исчезает (сообщение редактируется)
5. Система отвечает: ✅ Запись подтверждена

### 5.4 Отмена записи в Telegram

Нажать кнопку **Отменить** в Telegram (в сообщении новой заявки):

1. Запись удаляется из очереди подтверждений
2. Слот возвращается в статус Available
3. Клиент **не** получает email об отмене (запись не была подтверждена)

**Примечание:** это отличается от отмены уже подтверждённой записи, которая приходит клиентом через email ссылку.

## 6. Google Calendar

### 6.1 Синхронизация

Каждая студия имеет свой Google Calendar:

- **Rishon LeZion** → WOVSDH Rishon LeZion calendar
- **Ashdod** → WOVSDH Ashdod calendar

### 6.2 Когда создаются события?

События создаются при **подтверждении запись в Telegram боте**, не при создании записи.

**События содержат:**

```
Название: Анна Иванова
Время: 1 июня 2025, 08:00 - 09:00
Описание:
  Телефон: +972501234567
  Email: anna@example.com
```

### 6.3 Синхронизация в Google Calendar

Открыть [calendar.google.com](https://calendar.google.com) и выбрать календарь студии.

События синхронизируются автоматически (в реальном времени).

### 6.4 Если события не появляются

Проверить:

1. Правильны ли Google Calendar credentials в `.env`?
2. Refresh token не истёк ли?
3. Посмотреть логи сервера при подтверждении

## 7. Email уведомления

### 7.1 Автоматические письма

При подтверждении записи клиент получает письмо:

**Предмет:** Подтверждение записи в WOVSDH Nails

**Содержимое:**
- Данные студии и время
- .ics файл для добавления в календарь
- Ссылка на Google Calendar
- Ссылка для отмены записи

### 7.2 Отмена записи

Клиент может отменить запись через:

1. **Email ссылка** (кнопка в письме подтверждения)
2. **Сайт** (страница `/cancel?token=...`)

При отмене клиент получает письмо об отмене.

### 7.3 Если письма не отправляются

Проверить в Supabase:

```sql
SELECT * FROM email_logs WHERE error IS NOT NULL;
```

Посмотреть колонку `error` для описания проблемы.

## 9. Советы и best practices

### 9.1 Создание слотов

- **Создавайте слоты заранее** (за неделю или месяц)
- **Учитывайте выходные дни** (не создавайте слоты на выходные)
- **Не забывайте про обеденный перерыв** (нужно вручную блокировать в БД)

### 9.2 Управление записями

- **Регулярно проверяйте Telegram** (новые заявки приходят туда)
- **Подтверждайте записи максимум в день** (клиент ждёт подтверждения)
- **Отправляйте SMS напоминание** (не встроено, но рекомендуется)

### 9.3 Мониторинг

- **Проверяйте Email Logs в Supabase** (нет ли ошибок отправки)
- **Проверяйте Google Calendar** (события создаются правильно ли)
- **Обновляйте контактные данные** (номер, адрес, часы работы)

## 10. Troubleshooting

### Форма создания слотов не работает

**Проверить:**

1. Выбрана ли студия?
2. Правильный ли формат даты (YYYY-MM-DD)?
3. Время начала меньше времени окончания?
4. Открыта ли консоль браузера (F12) для ошибок?

### Слот не удаляется

Попробовать:

1. Перезагрузить страницу (F5)
2. Проверить интернет соединение
3. Попробовать через Supabase Dashboard напрямую

### Telegram сообщение не приходит

Проверить:

1. Добавлен ли сотрудник в `allowed_users` в Supabase?
2. `is_active = true`?
3. Правильный ли telegram_chat_id?
4. Вебхук установлен ли (`getWebhookInfo` — status OK)?

### Email не приходит

Проверить:

1. Email Logs в Supabase на наличие ошибок
2. SMTP переменные установлены ли?
3. Не в SPAM ли письмо?

## 11. Часто задаваемые вопросы

**Q: Могу ли я удалить запись клиента?**

A: Нет, прямого функционала нет. Это можно сделать через Supabase Dashboard (Admin > Table > bookings > select row > delete). При удалении запись клиента потеряется, но слот НЕ освободится. Лучше отменить запись через /cancel?token=... (если есть токен) или обновить статус в Supabase напрямую.

**Q: Как заблокировать слот на обеденный перерыв?**

A: Есть два варианта:

1. Создать слоты с длительностью совпадающей с перерывом (например, 60 мин) и удалить их. Это создаст "дыру" в расписании.
2. Вручную обновить статус в Supabase (SQL выше в разделе "Создание блокированных слотов").

**Q: Можно ли редактировать время записи?**

A: Нет, прямого функционала нет. Необходимо:

1. Отменить текущую запись (клиент получит email)
2. Создать новую запись на новое время

**Q: Кто может видеть админ-панель?**

A: Только люди, знающие пароль (`ADMIN_SECRET_KEY`). Панель защищена от публичного доступа.
